# v3.2.0 - 2026-06-07

## 不兼容變更

1. Carbon 組件的不兼容變更
    - `Carbon::createFromTimestamp()` 在 `v3.0` 中不再通過 `date_default_timezone_get()` 讀取默認時區。
    - `Carbon::diffInDays()` 默認情況下不再返回絕對值。請使用 `$absolute` 參數來控制結果是否為絕對值。

2. [#7625](https://github.com/hyperf/hyperf/pull/7625) 為了術語一致性，將 async-queue 組件中的 `JobInterface::getQueueName()` 重命名為 `getPoolName()`。

```php
<?php
// 變更前
class CustomJob extends \Hyperf\AsyncQueue\Job
{
    public function getQueueName(): string
    {
        return 'custom';
    }
}

// 變更後
class CustomJob extends \Hyperf\AsyncQueue\Job
{
    public function getPoolName(): string
    {
        return 'custom';
    }
}
```

```php
<?php

use Carbon\Carbon;

$t = time();

# 不兼容的用法
Carbon::createFromTimestamp($t, date_default_timezone_get());

# 正確的用法
Carbon::createFromTimestamp($t, date_default_timezone_get());
```

3. `logger` 配置結構已變更。請參考 [#7563](https://github.com/hyperf/hyperf/pull/7563)。

```php
<?php

// 變更前
return [
    'default' => [
        'driver' => 'daily',
        'path' => BASE_PATH . '/runtime/logs/hyperf.log',
        'level' => env('LOG_LEVEL', 'debug'),
        'days' => 7,
    ],
];

// 變更後
return [
    'default' => 'default',
    'channels' => [
        'default' => [
            'driver' => 'daily',
            'path' => BASE_PATH . '/runtime/logs/hyperf.log',
            'level' => env('LOG_LEVEL', 'debug'),
            'days' => 7,
        ],
        // 在此處添加你自定義的 channel
    ],
];
```

4. `cache` 配置結構已變更。請參考 [#7594](https://github.com/hyperf/hyperf/pull/7594)。

```php
<?php
// 變更前
return [
    'default' => [
        'driver' => RedisDriver::class,
        'packer' => PhpSerializerPacker::class,
        'prefix' => 'c:',
    ],
];

// 變更後
return [
    'default' => env('CACHE_DRIVER', 'default'),
    'stores' => [
        'default' => [
            'driver' => RedisDriver::class,
            'packer' => PhpSerializerPacker::class,
            'prefix' => 'c:',
        ],
    ],
];
```

5. 使用 `swow` 時新增依賴。

`config/autoload/dependencies.php`

```php
<?php
use Hyperf\Contract\ResponseEmitterInterface;
use Hyperf\Engine\ResponseEmitter;

return [
    ResponseEmitterInterface::class => ResponseEmitter::class,
];
```

## 注意

1. 如果你已經升級到 PHP 8.4，請注意 `fgetcsv` 和 `fputcsv` 方法現在都需要為 `escape` 參數提供默認值。

```
fputcsv($fp, $fields) 必須修改為 fputcsv($fp, $fields, escape: '')
fgetcsv($fp) 必須修改為 fgetcsv($fp, escape: '')
```

## 依賴升級

- 升級 php 版本至 `>=8.2`
- 升級 `elasticsearch/elasticsearch` 版本至 `^8.0 || ^9.0`
- 升級 `nikic/php-parser` 版本至 `5.6`
- 升級 `symfony/*` 組件至 `^6.0 || ^7.0`
- 升級 `phpunit/phpunit` 版本至 `^11.0`
- 升級 `google/protobuf` 版本至 `^3.6.1 || ^4.2`。
- 升級 `guzzlehttp/guzzle` 版本至 `^7.0`。

## 移除

- [#7278](https://github.com/hyperf/hyperf/pull/7278) 移除已廢棄的 `laminas/laminas-mime` 包。
- [#7573](https://github.com/hyperf/hyperf/pull/7573) 移除已廢棄的 `Hyperf\Serializer\Contract\CacheableSupportsMethodInterface` 接口。

- [#7620](https://github.com/hyperf/hyperf/pull/7620) 新增 Symfony 7.4 兼容性，支持批量命令註冊。
