# 2.1 Upgrade Guide

- Version 2.1 mainly adds the underlying driver [hyperf/engine](https://github.com/hyperf/engine), allowing the `Hyperf` framework to run on `Swoole` or `Swow`.
- The minimum PHP version is modified from 7.2 to 7.3.

> Swow is currently a preview version, please use it with caution.

## Modify Hyperf Component Versions

Directly change all `hyperf/*` in `composer.json` to `2.1.*`.

```json
{
    "require": {
        "php": ">=7.3",
        "ext-json": "*",
        "ext-openssl": "*",
        "ext-pdo": "*",

        "ext-redis": "*",

        "hyperf/cache": "2.1.*",
        "hyperf/command": "2.1.*",
```

After that, just execute `composer update -o` to complete the upgrade normally.

## Add optimize-autoloader Configuration

Because `Hyperf` has required the `class_map` generated by `composer` since `2.0`, this requires users to use `-o` for optimization every time dependencies are updated, but many users have never had this habit.
Therefore, we suggest adding corresponding configuration in `composer.json` to meet this need.

```json
{
    "config": {
        "optimize-autoloader": true,
        "sort-packages": true
    }
}
```

## Modify SwooleEvent

The class `Hyperf\Server\SwooleEvent` has been renamed to `Hyperf\Server\Event` in `2.1`, so we need to modify the corresponding code in the `server.php` configuration.

> SwooleEvent will be officially removed in 3.0, please change it to Event as soon as possible.

```php
<?php

declare(strict_types=1);

use Hyperf\Server\Event;
use Hyperf\Server\Server;

return [
    'mode' => SWOOLE_BASE,
    'servers' => [
        [
            'name' => 'http',
            'type' => Server::SERVER_HTTP,
            'host' => '0.0.0.0',
            'port' => 9501,
            'sock_type' => SWOOLE_SOCK_TCP,
            'callbacks' => [
                Event::ON_REQUEST => [Hyperf\HttpServer\Server::class, 'onRequest'],
            ],
        ],
    ],
    'callbacks' => [
        Event::ON_BEFORE_START => [Hyperf\Framework\Bootstrap\ServerStartCallback::class, 'beforeStart'],
        Event::ON_WORKER_START => [Hyperf\Framework\Bootstrap\WorkerStartCallback::class, 'onWorkerStart'],
        Event::ON_PIPE_MESSAGE => [Hyperf\Framework\Bootstrap\PipeMessageCallback::class, 'onPipeMessage'],
        Event::ON_WORKER_EXIT => [Hyperf\Framework\Bootstrap\WorkerExitCallback::class, 'onWorkerExit'],
    ],
];
```

## Paginator

Because the `hyperf/paginator` component has been removed from the `hyperf/database` dependency. Students who use the paginator in database need to additionally introduce the `hyperf/paginator` component.

## Modify DBAL Version

If you use the `doctrine/dbal` component, you need to upgrade to version `^3.0`.

## Remove doctrine/common component

There is a dependency conflict between the `doctrine/common` component and `hyperf/utils`. Therefore, you need to remove this component from `composer.json`.

```bash
# Remove component
composer remove doctrine/common
# Update
composer update "hyperf/*" -o
```

## Precautions

- Try not to modify the engine of old projects to Swow. If you want to use Swow, please try it in new projects. Because Swow is not a substitute for Swoole, not all Swoole scenarios can find corresponding alternative solutions in Swow.
