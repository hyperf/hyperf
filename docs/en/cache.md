# Cache

[hyperf/cache](https://github.com/hyperf/cache) provides aspect-oriented caching implemented based on `Aspect`, and also provides cache classes that implement `Psr\SimpleCache\CacheInterface`.

## Installation

```
composer require hyperf/cache
```

## Default Configuration

| Configuration | Default Value | Remarks |
|:------:|:----------------------------------------:|:---------------------:|
| driver | Hyperf\Cache\Driver\RedisDriver | Cache driver, defaults to Redis |
| packer | Hyperf\Codec\Packer\PhpSerializerPacker | Packer |
| prefix | c: | Cache prefix |
| skip_cache_results | [] | Specified results not cached |

```php
<?php

return [
    'default' => [
        'driver' => Hyperf\Cache\Driver\RedisDriver::class,
        'packer' => Hyperf\Codec\Packer\PhpSerializerPacker::class,
        'prefix' => 'c:',
        'skip_cache_results' => [],
    ],
];
```

## Usage

### Simple Cache Mode

Simple Cache refers to the [PSR-16](https://www.php-fig.org/psr/psr-16/) specification. This component adapts to this specification. If you wish to use a cache class that implements `Psr\SimpleCache\CacheInterface`, for example, to override the cache module of `EasyWeChat`, you can directly obtain `Psr\SimpleCache\CacheInterface` from the dependency injection container, as shown below:

```php

$cache = $container->get(\Psr\SimpleCache\CacheInterface::class);

```

### Annotation Mode

The component provides the `Hyperf\Cache\Annotation\Cacheable` annotation, which acts on class methods and can configure corresponding cache prefixes, expiration times, listeners, and cache groups.
For example, `UserService` provides a `user` method, which can query user information for a corresponding `id`. After adding the `Hyperf\Cache\Annotation\Cacheable` annotation, the corresponding Redis cache will be automatically generated, with the `key` value being `user:id` and the timeout being `9000` seconds. When querying for the first time, it will be queried from the database; for subsequent queries, it will be queried from the cache.

```php
<?php

namespace App\Services;

use App\Models\User;
use Hyperf\Cache\Annotation\Cacheable;

class UserService
{
    #[Cacheable(prefix: "user", ttl: 9000, listener: "user-update")]
    public function user($id)
    {
        $user = User::query()->where('id',$id)->first();

        if($user){
            return $user->toArray();
        }

        return null;
    }
}
```

### Clearing Cache Generated by `#[Cacheable]`

We provide two annotations, `CachePut` and `CacheEvict`, to implement cache updating and cache clearing operations.

Of course, we can also delete the cache via events. Below, create a new `Service` and provide a method to help us handle the cache.

> However, we recommend that users use annotation processing rather than listeners.

```php
<?php

declare(strict_types=1);

namespace App\Service;

use Hyperf\Di\Annotation\Inject;
use Hyperf\Cache\Listener\DeleteListenerEvent;
use Psr\EventDispatcher\EventDispatcherInterface;

class SystemService
{
    #[Inject]
    protected EventDispatcherInterface $dispatcher;

    public function flushCache($userId)
    {
        $this->dispatcher->dispatch(new DeleteListenerEvent('user-update', [$userId]));

        return true;
    }
}
```

When we customize the `value` of `Cacheable`, for example, in the following situation.

```php
<?php

declare(strict_types=1);

namespace App\Service\Cache;

use Hyperf\Cache\Annotation\Cacheable;

class DemoService
{

    #[Cacheable(prefix: "cache", value: "_#{id}", listener: "user-update")]
    public function getCache(int $id)
    {
        return $id . '_' . uniqid();
    }
}
```

Then you need to modify the `$arguments` variable in the `DeleteListenerEvent` constructor accordingly. The specific code is as follows.

```php
<?php

declare(strict_types=1);

namespace App\Service;

use Hyperf\Di\Annotation\Inject;
use Hyperf\Cache\Listener\DeleteListenerEvent;
use Psr\EventDispatcher\EventDispatcherInterface;

class SystemService
{
    #[Inject]
    protected EventDispatcherInterface $dispatcher;

    public function flushCache($userId)
    {
        $this->dispatcher->dispatch(new DeleteListenerEvent('user-update', ['id' => $userId]));

        return true;
    }
}
```

## Annotation Introduction

### Cacheable

For example, in the following configuration, the cache prefix is `user`, the timeout is `7200`, and the deletion event name is `USER_CACHE`. The generated corresponding cache `KEY` is `c:user:1`.

```php
<?php

declare(strict_types=1);

namespace App\Service;

use App\Models\User;
use Hyperf\Cache\Annotation\Cacheable;

class UserService
{
    #[Cacheable(prefix: "user", ttl: 7200, listener: "USER_CACHE")]
    public function user(int $id): array
    {
        $user = User::query()->find($id);

        return [
            'user' => $user->toArray(),
            'uuid' => $this->unique(),
        ];
    }
}
```

After setting `value`, the framework will name the cache `KEY` according to the set rules. As in the following instance, when `$user->id = 1`, the cache `KEY` is `c:userBook:_1`

> This configuration also supports other types of cache annotations mentioned below.

```php
<?php

declare(strict_types=1);

namespace App\Service;

use App\Models\User;
use Hyperf\Cache\Annotation\Cacheable;

class UserBookService
{
    #[Cacheable(prefix: "userBook", ttl: 6666, value: "_#{user.id}")]
    public function userBook(User $user): array
    {
        return [
            'book' => $user->book->toArray(),
            'uuid' => $this->unique(),
        ];
    }
}
```

### CacheAhead

For example, in the following configuration, the cache prefix is `user`, the timeout is `7200`, and the generated corresponding cache `KEY` is `c:user:1`. And between 7200 - 600 seconds, cache initialization is performed every 10 seconds until it succeeds for the first time.

```php
<?php

declare(strict_types=1);

namespace App\Service;

use App\Models\User;
use Hyperf\Cache\Annotation\CacheAhead;

class UserService
{
    #[CacheAhead(prefix: "user", ttl: 7200, aheadSeconds: 600, lockSeconds: 10)]
    public function user(int $id): array
    {
        $user = User::query()->find($id);

        return [
            'user' => $user->toArray(),
            'uuid' => $this->unique(),
        ];
    }
}
```

### CachePut

`CachePut` is different from `Cacheable`, it executes the function body each time it is called, and then overwrites the cache. So when we want to update the cache, we can call the relevant method.

```php
<?php

declare(strict_types=1);

namespace App\Service;

use App\Models\User;
use Hyperf\Cache\Annotation\CachePut;

class UserService
{
    #[CachePut(prefix: "user", ttl: 3601)]
    public function updateUser(int $id)
    {
        $user = User::query()->find($id);
        $user->name = 'HyperfDoc';
        $user->save();

        return [
            'user' => $user->toArray(),
            'uuid' => $this->unique(),
        ];
    }
}
```

### CacheEvict

CacheEvict is easier to understand. After executing the method body, it will actively clear the cache.

```php
<?php

declare(strict_types=1);

namespace App\Service;

use Hyperf\Cache\Annotation\CacheEvict;

class UserBookService
{
    #[CacheEvict(prefix: "userBook", value: "_#{id}")]
    public function updateUserBook(int $id)
    {
        return true;
    }
}
```

## Cache Driver

### Redis Driver

`Hyperf\Cache\Driver\RedisDriver` will store cached data in `Redis`, requiring the user to configure the corresponding `Redis configuration`. This mode is the default mode.

### Process Memory Driver

If you need to cache data in memory, you can try this driver.

The configuration is as follows:

```php
<?php

return [
    'memory' => [
        'driver' => Hyperf\Cache\Driver\MemoryDriver::class,
    ],
];
```

### Coroutine Memory Driver

If you need to cache data in `Context`, you can try this driver. For example, in the following application scenario, `Demo::get` will be called multiple times in multiple places, but you do not want to query `Redis` every time.

```php
<?php
use Hyperf\Cache\Annotation\Cacheable;

class Demo
{    
    public function get($userId, $id)
    {
        return $this->getArray($userId)[$id] ?? 0;
    }

    #[Cacheable(prefix: "test", group: "co")]
    public function getArray(int $userId): array
    {
        return $this->redis->hGetAll($userId);
    }
}
```

The corresponding configuration is as follows:

```php
<?php

return [
    'co' => [
        'driver' => Hyperf\Cache\Driver\CoroutineMemoryDriver::class,
        'packer' => Hyperf\Codec\Packer\PhpSerializerPacker::class,
    ],
];
```
