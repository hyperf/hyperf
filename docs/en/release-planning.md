# Release Planning

## Version Lifecycle

| Version | Status | Active Support End Date | Security Maintenance End Date | Release or Estimated Release Date |
| :--- | :--- | :--- | :--- | :--- |
| 3.1(LTS) | Active Support | 2026-01-01 | 2027-01-01 | 2023-12-01 |
| 3.0 | Security Support | 2023-11-30 | 2024-06-30 | 2023-01-03 |
| 2.2 | End of Life | 2022-06-20 | 2023-11-30 | 2021-07-19 |
| 2.1 | End of Life | 2021-06-30 | 2021-12-31 | 2020-12-28 |
| 2.0 | End of Life | 2020-12-28 | 2021-06-30 | 2020-06-22 |
| 1.1 | End of Life | 2020-06-23 | 2020-12-31 | 2019-10-08 |
| 1.0 | End of Life | 2019-10-08 | 2019-12-31 | 2019-06-20 |

* Active support includes bug fixes, security fixes, feature iterations, and new features within the regular iteration cycle;
* Security maintenance only includes security fixes;
* Versions in End-of-Life status will no longer receive any code changes. Please follow the upgrade guide to upgrade to the latest version as soon as possible for better support.

## Version Iteration Cycle

Hyperf adopts an agile development model, with an iteration plan every week, and releases a version `every Monday (UTC/GMT+08:00)`. Generally, it is a z-version release, or it could be a y-version. For x-versions, specific iteration plans and timelines are decided based on actual research findings.
For the version rules adopted by Hyperf, please refer to the [Version Description](versions.md) chapter.
