# Guia de atualização 2.2

A versão 2.2 adiciona principalmente a adaptação para `PHP 8` e suporte a anotações nativas.

## Modificar versão dos componentes Hyperf

Basta alterar `hyperf/*` no `composer.json` para `2.2.*`.

> hyperf/engine não segue o número de versão do framework, então não é necessário modificar

Além disso, podemos executar `composer require "hyperf/ide-helper:2.2.*" --dev` para instalar `hyperf/ide-helper`. Este componente pode ajudar a sugerir os parâmetros que anotações podem definir ao usar anotações nativas.

Depois disso, basta executar `composer update -o`, e a atualização pode ser concluída normalmente.

## Modificar o script de teste unitário

Foi adicionada a opção `--prepend test/bootstrap.php`.

```json
{
    "scripts": {
        "test": "co-phpunit --prepend test/bootstrap.php -c phpunit.xml --colors=always"
    }
}
```

## Instalar a extensão pcntl

A nova versão do scanning de anotações usa a extensão `pcntl`, então certifique-se de que seu `PHP` tem essa extensão instalada.

```shell
php --ri pcntl

pcntl

pcntl support => enabled
```

> Ao habilitar `grpc`, você precisa adicionar `grpc.enable_fork_support= 1;` ao `php.ini` para dar suporte à abertura de processos filhos.

## AMQP

> Atenção a usuários que usam AMQP: esta seção pode ser ignorada se você não usa.

Como o componente `AMQP` foi atualizado amplamente e passou a suportar multiplexing, existem mudanças na configuração. Siga a configuração mais recente abaixo e modifique conforme apropriado.

```php
<?php

return [
    'default' => [
        'host' => env('AMQP_HOST', 'localhost'),
        'port' => (int) env('AMQP_PORT', 5672),
        'user' => env('AMQP_USER', 'guest'),
        'password' => env('AMQP_PASSWORD', 'guest'),
        'vhost' => env('AMQP_VHOST', '/'),
        'concurrent' => [
            'limit' => 1,
        ],
        'pool' => [
            // Número de conexões abertas ao mesmo tempo
            // Como a nova versão da conexão suporta multiplexing, é possível alcançar alta concorrência com um número muito pequeno de conexões
            'connections' => 2,
        ],
        'params' => [
            'insist' => false,
            'login_method' => 'AMQPLAIN',
            'login_response' => null,
            'locale' => 'en_US',
            'connection_timeout' => 3,
            'read_write_timeout' => 6,
            'context' => null,
            'keepalive' => true,
            'heartbeat' => 3,
            'channel_rpc_timeout' => 0.0,
            'close_on_destruct' => false,
            // O valor máximo de canais ociosos no multiplexing. Após este número ser excedido, os canais excedentes serão fechados.
            'max_idle_channels' => 10,
        ],
    ],
];

```

## Configuration Center

> Usuários que usam Configuration Center: note que esta seção pode ser ignorada se você não usa.

O configuration center foi totalmente refatorado nesta versão; certifique-se de reler a documentação correspondente com cuidado.

O Unity precisa introduzir o componente `hyperf/config-center`. O comando é:

```shell
composer require "hyperf/config-center:~2.2.0"
```

E introduza os componentes dependentes do driver conforme o driver usado. Se você usa `Apollo`, precisa importar o componente `hyperf/config-apollo`, e os demais drivers são similares.

Ao mesmo tempo, todas as informações de configuração relacionadas ao configuration center foram reunidas em `config/autoload/config_center.php`. Configure conforme a nova estrutura. Se este arquivo não existir, você pode executar o comando `php bin/hyperf.php vendor:publish hyperf/config-center` para criar.

## Service Center

Para usuários que usam o componente `hyperf/service-gonvernace`, como o adapter `consul` foi removido deste componente, a nova versão precisa adicionar o componente `hyperf/service-governance-consul`. O comando é:

```shell
composer require "hyperf/service-governance-consul:~2.2.0"
```

Usuários que usam `nacos` como driver de service center precisam introduzir o componente `hyperf/service-governance-nacos`. O comando é:

```shell
composer require "hyperf/service-governance-nacos:~2.2.0"
```

## php-cs-fixer

Se você não precisa fazer upgrade do `php-cs-fixer` para a versão `3.0`, pode ignorar esta seção.

1. Modificar o número da versão

```
"friendsofphp/php-cs-fixer": "^3.0"
```

2. Renomear o arquivo .php_cs

Renomeie para `.php-cs-fixer.php` e modifique o código correspondente de acordo com os seguintes registros de mudança:

```diff
- return PhpCsFixer\Config::create()
+ return (new PhpCsFixer\Config())

- 'commentType' => 'PHPDoc',
+ 'comment_type' => 'PHPDoc',
```

## A versão PHP 7.3 diminuiu a compatibilidade do DI

> Usuários de versões mais novas do PHP podem ignorar esta seção.

Nas versões `2.0` - `2.1`, para implementar `AOP` aplicando em objetos não gerenciados pelo `DI` (como objetos instanciados pela palavra-chave `new`), a implementação subjacente usa o componente `BetterReflection`. Isso trouxe uma nova experiência, mas também alguns problemas difíceis de contornar, como:

- inicialização lenta do projeto sem scan cache
- `Inject` e `Value` não têm efeito em casos especiais
- `BetterReflection` ainda não suporta PHP 8 (até a release 2.2)

Na nova versão, o uso de `BetterReflection` foi descontinuado, e o método de `subprocess scanning` foi adotado para resolver os pontos acima. Porém, também existem algumas incompatibilidades em versões mais baixas do `PHP`:

Após iniciar a aplicação com `PHP 7.3`, eu obtenho um erro semelhante ao seguinte:

```bash
PHP Fatal error: Interface 'Hyperf\Signal\SignalHandlerInterface' not found in vendor/hyperf/process/src/Handler/ProcessStopHandler.php on line 17

PHP Fatal error: Interface 'Symfony\Component\Serializer\SerializerInterface' not found in vendor/hyperf/utils/src/Serializer/Serializer.php on line 46
````

Esse problema acontece porque, no `PHP 7.3`, a reflexão é obtida por `subprocess scanning`, e quando uma classe implementa uma `Interface` que não existe, isso dispara uma exceção `Interface not found`. Em versões mais altas do `PHP`, isso não acontece.

A solução é criar a `Interface` correspondente e importá-la normalmente. Para o erro acima, a solução é instalar os componentes dependentes correspondentes.

> Claro, é melhor fazer upgrade para 7.4 ou 8.0

```bash
composer require hyperf/signal

composer require symfony/serializer
```

## File system

> Para usuários que usam o file system: note que esta seção pode ser ignorada se você não usa.

A partir do v2.2, o Hyperf passará a suportar os componentes `League\Flysystem` tanto em `v1.0` quanto em `v2.0`.

Se você usa `League\Flysystem` `v2.0`, os adapters correspondentes também precisam ser ajustados de acordo com a lista a seguir. Consulte o capítulo `File system` para a documentação específica.

- Alibaba Cloud OSS adapter

```shell
composer require hyperf/flysystem-oss
````

- S3 adapter

```shell
composer require "league/flysystem-aws-s3-v3:^2.0"
composer require hyperf/guzzle
```

- Qiniu Adapter

```shell
composer require "overtrue/flysystem-qiniu:^2.0"
```

- memory adapter

```shell
composer require "league/flysystem-memory:^2.0"
```

- Tencent Cloud COS Adapter

> A configuração básica deste adapter é ligeiramente diferente da versão `v2.0` do componente `overtrue/flysystem-cos`; faça modificações apropriadas conforme a configuração mais recente.

```shell
composer require "overtrue/flysystem-cos:^4.0"
```
