# Guia de atualização 2.1

- A versão 2.1 adiciona principalmente o driver subjacente [hyperf/engine](https://github.com/hyperf/engine), permitindo que o framework `Hyperf` rode por cima de `Swoole` ou `Swow`.
- A versão mínima do PHP mudou de 7.2 para 7.3

> Swow é temporariamente uma versão de preview; use com cautela.

## Adicionar configuração optimize-autoloader

Como o `Hyperf` precisa usar o `class_map` gerado pelo `composer` desde a versão `2.0`, isso exige que usuários usem `-o` para otimização toda vez que atualizam dependências, mas muitos usuários não têm esse hábito.
Por isso, recomendamos adicionar a configuração correspondente ao `composer.json` para atender a essa necessidade.

```json
{
    "config": {
        "optimize-autoloader": true,
        "sort-packages": true
    }
}
```

## Modificar SwooleEvent

A classe `Hyperf\Server\SwooleEvent` foi renomeada para `Hyperf\Server\Event` no `2.1`, então precisamos modificar o código correspondente na configuração `server.php`.

> SwooleEvent será removido oficialmente no 3.0; mude para Event o quanto antes

```php
<?php

declare(strict_types=1);

use Hyperf\Server\Event;
use Hyperf\Server\Server;

return [
    'mode' => SWOOLE_BASE,
    'servers' => [
        [
            'name' => 'http',
            'type' => Server::SERVER_HTTP,
            'host' => '0.0.0.0',
            'port' => 9501,
            'sock_type' => SWOOLE_SOCK_TCP,
            'callbacks' => [
                Event::ON_REQUEST => [Hyperf\HttpServer\Server::class, 'onRequest'],
            ],
        ],
    ],
    'callbacks' => [
        Event::ON_BEFORE_START => [Hyperf\Framework\Bootstrap\ServerStartCallback::class, 'beforeStart'],
        Event::ON_WORKER_START => [Hyperf\Framework\Bootstrap\WorkerStartCallback::class, 'onWorkerStart'],
        Event::ON_PIPE_MESSAGE => [Hyperf\Framework\Bootstrap\PipeMessageCallback::class, 'onPipeMessage'],
        Event::ON_WORKER_EXIT => [Hyperf\Framework\Bootstrap\WorkerExitCallback::class, 'onWorkerExit'],
    ],
];
```

## Paginator

Como o componente `hyperf/paginator` foi removido da dependência `hyperf/database`, quem usa paginação no database também precisa introduzir o componente `hyperf/paginator`.

## Modificar versão do DBAL

Se você usa o componente `doctrine/dbal`, precisa fazer upgrade para a versão `^3.0`.

## Remover o componente doctrine/common

O componente `doctrine/common` tem conflito de dependência com `hyperf/utils`. Então este componente precisa ser removido do `composer.json`.

```bash
# remover componente
composer remove doctrine/common
# atualizar
composer update "hyperf/*" -o
```

## Precauções

- Tente não mudar o engine do projeto antigo para Swow. Se você quiser usar Swow, experimente em um novo projeto. Como Swow não é um substituto do Swoole, nem todos os cenários do Swoole têm alternativas correspondentes em Swow.
