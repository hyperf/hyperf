Misalnya, di v1.x, Anda hanya dapat menggunakan AOP dalam instance kelas yang dibuat oleh Hyperf DI container, Anda tidak dapat memotong instance kelas yang dibuat oleh pengidentifikasi `new`. Namun sekarang, di v2.0, hal tersebut tersedia. Tetapi masih ada pengecualian, kelas-kelas yang digunakan pada tahap bootstrap masih tidak dapat berfungsi.
    2. Di v1.x, AOP HANYA tersedia untuk kelas normal, bukan untuk kelas Final yang tidak dapat diwarisi oleh subclass. Namun sekarang, di v2.0, hal tersebut tersedia.
    3. Di v1.x, Anda tidak dapat menggunakan nilai properti yang ditandai oleh anotasi `@Inject` atau `@Value` di konstruktor kelas saat ini. Namun sekarang, di v2.0, hal tersebut tersedia.
    4. Di v1.x, Anda hanya dapat menggunakan anotasi `@Inject` dan `@Value` di instance kelas yang dibuat oleh Hyperf DI container. Namun sekarang, di v2.0, hal itu tersedia dengan cara apa pun, seperti instance kelas yang dibuat oleh pengidentifikasi `new`.
    5. Di v1.x, Anda harus mendefinisikan namespace penuh dari kelas Anotasi saat Anda menggunakan Anotasi tersebut. Namun sekarang, di v2.0, komponen ini menyediakan mekanisme import global, Anda dapat mendefinisikan alias untuk Anotasi untuk menggunakan Anotasi secara langsung tanpa menggunakan namespace. Misalnya, Anda dapat mendefinisikan anotasi `@Inject` di kelas mana pun tanpa mendefinisikan `use Hyperf\Di\Annotation\Inject;`.
    6. Di v1.x, kelas proxy yang dibuat oleh container DI adalah subclass dari target class, mekanisme ini akan menyebabkan konstanta ajaib mengembalikan nilai dari kelas proxy tetapi bukan kelas asli, seperti `__CLASS__`. Namun sekarang, di v2.0, kelas proxy akan mempertahankan struktur yang sama dengan kelas asli, tidak akan mengubah nama kelas atau struktur kelas.
    7. Di v1.x, kelas proxy tidak akan dibuat ulang ketika file proxy ada meskipun kode kelas proxy diubah, strategi ini akan meningkatkan waktu pemindaian, tetapi pada saat yang sama, ini akan menyebabkan tingkat ketidaknyamanan pengembangan tertentu. Dan sekarang, di v2.0, file cache dari kelas proxy akan dibuat berdasarkan konten kode dari kelas proxy, perubahan ini akan mengurangi beban mental saat pengembangan.
    8. Menambahkan parameter `priority` untuk Aspect, sekarang Anda dapat mendefinisikan `priority` di kelas Aspect melalui properti kelas atau properti anotasi, untuk mengatur urutan aspek.
    9. Di v1.x, Anda hanya dapat mendefinisikan kelas Aspect dengan anotasi `@Aspect`, Anda tidak dapat mendefinisikan kelas Aspect melalui file konfigurasi. Namun sekarang, di v2.0, hal itu tersedia untuk mendefinisikan kelas Aspect melalui file konfigurasi atau ConfigProvider.
    10. Di v1.x, Anda harus menambahkan `Hyperf\Di\Listener\LazyLoaderBootApplicationListener` untuk mengaktifkan lazy loading. Di 2.0, lazy loading dapat digunakan secara langsung. Listener ini karena itu dihapus.
    11. Menambahkan konfigurasi `annotations.scan.class_map`, sekarang Anda dapat mengganti konten kelas mana pun secara dinamis di atas aturan autoload.
