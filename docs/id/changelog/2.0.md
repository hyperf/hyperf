# v2.0.0 - 2020-06-22

## Perubahan Utama

1. Refactor [hyperf/di](https://github.com/hyperf/di) komponen, di dalam
particular, AOP dan Annotation Scanner are Mengoptimalkan, di dalam v2.0, the
komponen use a brand new loading mechanism ke provided an incredible AOP
function.
1. The most significant functional differences compared ke v1.x is that you can
cut ke dalam any classes di dalam any ways dengan Aspect. untuk example, di
dalam v1.x, you can only use AOP di dalam the class instance that created oleh
Hyperf DI container, you tidak cut ke dalam the class instance that created oleh
`new` identifier. But now, di dalam v2.0, it is available. But there is still
has an exception, the classes that used di dalam bootstrap stage still tidak
works.
2. di dalam v1.x, the AOP ONLY available untuk the normal classes, not untuk
Final class that tidak be inherited oleh a subclass. But now, di dalam v2.0. it
is available.
3. di dalam v1.x, you tidak use the property value that marked oleh `@Inject`
or `@Value` annotation di dalam the constructor dari current class. But now, di
dalam v2.0, it is available.
4. di dalam v1.x, you can only use `@Inject` dan `@Value` annotation di dalam
the class instance that created oleh Hyperf DI container. But now, di dalam
v2.0, it is available di dalam any ways, such sebagai the class instance that
created oleh `new` identifier.
5. di dalam v1.x, you have untuk mendefinisikan the full namespace dari
Annotation class ketika you use the Annotation. But now, di dalam v2.0, the
komponen provide a global import mechanism, you cloud define an alias untuk
Annotation ke use the Annotation secara langsung without menggunakan the
namespace. untuk example, you cloud define `@Inject` annotation di dalam any
class without define `use Hyperf\Di\Annotation\Inject;`.
6. di dalam v1.x, the proxy class that created oleh the DI container is a
subclass dari the target class, this mechanism akan cause the magic constant
akan return the value dari proxy class but not original class, such sebagai
`__CLASS__`. But now, di dalam v2.0, the proxy class akan keep the same
structure dengan the original class, akan not change the class name or the class
structure.
7. di dalam v1.x, the proxy class akan not re-generate ketika the proxy file
exists even the code dari the proxy class Mengubah, this strategy akan improve
the time-consuming dari scan, but pada saat yang sama, this akan lead ke a
certain degree dari development inconvenience. dan now, di dalam v2.0, the file
cache dari proxy class akan generated according ke the code content dari the
proxy class, this changes akan reduces the mental burden dari development.
8. Add `priority` parameter untuk Aspect, now you could define `priority` pada
aspect class oleh class property or annotation property, ke manage the order
dari the aspects.
9. di dalam v1.x, you can only define an Aspect class oleh `@Aspect`
annotation, you tidak define the Aspect class oleh configuration file. But now,
di dalam v2.0, it is available untuk mendefinisikan the Aspect class oleh
configuration file or ConfigProvider.
10. di dalam v1.x, you have ke add
`Hyperf\Di\Listener\LazyLoaderBootApplicationListener` ke enable lazy loading.
di dalam 2.0, lazy loading can be used secara langsung. This listener is
therefore Menghapus.
11. Menambahkan `annotations.scan.class_map` configuration, now you could
replace any content dari class dynamically above the autoload rules.
