# 1.1 Upgrade Guide

Versi 1.1 menambahkan banyak fitur baru, tetapi beberapa perubahan juga melibatkan penyesuaian struktur Skeleton dan item konfigurasi. Jika Anda sudah menggunakan proyek untuk bisnis dan merupakan proyek 1.0 yang dibuat berdasarkan Skeleton resmi, Anda bisa menyesuaikan sesuai poin-poin berikut. Jika proyek baru, ikuti dokumentasi dan gunakan `composer create-project hyperf/hyperf-skeleton` untuk membuat proyek baru dengan struktur skeleton terbaru.

## Meningkatkan Swoole ke 4.4+

Versi 1.1 menaikkan persyaratan versi minimum Swoole dari 4.3+ menjadi 4.4+. Ada beberapa perbedaan detail penggunaan antara kedua versi ini, yang sudah diadaptasi oleh Hyperf di versi sebelumnya. Bagi pengguna Hyperf, tidak perlu khawatir dengan perbedaan tersebut. Kami menaikkan versi minimum Swoole terutama untuk mengurangi beban historis, dan Swoole 4.4 sebagai versi LTS (Long Term Support) juga berarti lebih stabil dan andal.

Hyperf akan melakukan deteksi versi Swoole saat memulai, tetapi untuk menyatukan batasan dependensi versi Swoole dengan lebih baik, kami sarankan mengubah kondisi dependensi Swoole di `composer.json` menjadi `"ext-swoole": ">=4.4"`.

## Menambahkan Konstanta SWOOLE_HOOK_FLAGS

Tambahkan satu baris definisi konstanta di file entry aplikasi `bin/hyperf.php` dan file entry untuk unit test `test/bootstrap.php` sebagai berikut:

```php
! defined('SWOOLE_HOOK_FLAGS') && define('SWOOLE_HOOK_FLAGS', SWOOLE_HOOK_ALL);
```

Referensi: [Referensi file entry](https://github.com/hyperf/hyperf-skeleton/blob/70062b7bbf29e23cda2f30680e02aa3b26ebd6f7/bin/hyperf.php#L11) [Referensi file entry unit test](https://github.com/hyperf/hyperf-skeleton/blob/70062b7bbf29e23cda2f30680e02aa3b26ebd6f7/test/bootstrap.php#L20)

## Memindahkan File config/dependencies.php dan Menyesuaikan Struktur File

Pindahkan `config/dependencies.php` → `config/autoload/dependencies.php`, dan hapus `dependencies` tingkat pertama di file konfigurasi, sebagai berikut:

Struktur file 1.0:
```php
<?php
// file config/dependencies.php

return [
    'dependencies' => [
        FooInterface::class => Foo::class
    ],
];
```

Struktur file 1.1:
```php
<?php
// file config/autoload/dependencies.php

return [
    FooInterface::class => Foo::class
];
```

## Menyesuaikan Konten File config/container.php

Karena versi 1.1 menyesuaikan lokasi dan struktur file `dependencies.php`, kita juga perlu menyesuaikan file `config/container.php` agar dependency injection container bisa berjalan dengan benar. Pada saat yang sama, kami juga menyediakan cara penulisan yang lebih sederhana untuk `config/container.php`. `DefinitionSourceFactory` menggabungkan banyak perilaku default. Anda tinggal mengubah konten file `config/container.php` menjadi berikut:

> Fungsi cache pemindaian Annotation diaktifkan secara default. Anda dapat memodifikasi parameter pertama dari input `DefinitionSourceFactory` untuk menonaktifkan fungsi ini.

```php
<?php
/**
 * Inisialisasi dependency injection container yang mengimplementasikan PSR-11 dan mengembalikan container.
 */
declare(strict_types=1);

use Hyperf\Di\Container;
use Hyperf\Di\Definition\DefinitionSourceFactory;
use Hyperf\Utils\ApplicationContext;
use Psr\Container\ContainerInterface;

$container = new Container((new DefinitionSourceFactory(true))());
if (! $container instanceof ContainerInterface) {
    throw new RuntimeException('Dependency injection container tidak valid.');
}
return ApplicationContext::setContainer($container);
```

## Menyesuaikan WebSocket Controller

Karena versi 1.1 menyesuaikan batasan parameter input dari `onMessage` dan `onOpen`, perlu dimodifikasi secara manual menjadi `Swoole\WebSocket\Server`. Kode spesifiknya adalah sebagai berikut:

```php
<?php
declare(strict_types=1);

namespace App\Controller;

use Hyperf\Contract\OnMessageInterface;
use Hyperf\Contract\OnOpenInterface;
use Swoole\Http\Request;
use Swoole\Websocket\Frame;
use Swoole\WebSocket\Server as WebSocketServer;

class WebSocketController implements OnMessageInterface, OnOpenInterface
{
    public function onMessage(WebSocketServer $server, Frame $frame): void
    {
    }

    public function onOpen(WebSocketServer $server, Request $request): void
    {
    }
}
```

## Menyesuaikan ConfigProvider dari Komponen Kustom

Di versi 1.0, `scan.path` disesuaikan menjadi `annotations.scan.path` di versi 1.1. Anda perlu memodifikasi kelas ConfigProvider dari semua komponen kustom untuk beradaptasi dengan perubahan ini. Jika komponen kustom Anda tidak melibatkan konfigurasi fungsional pemindaian Annotation, Anda dapat mengabaikan penyesuaian ini, seperti yang ditunjukkan di bawah ini:

Struktur file ConfigProvider 1.0:
```php
class ConfigProvider
{

    public function __invoke(): array
    {
        return [
            'scan' => [
                'paths' => [
                    __DIR__,
                ],
            ],
        ];
    }
}
```

Struktur file ConfigProvider 1.1:
```php
class ConfigProvider
{

    public function __invoke(): array
    {
        return [
            'annotations' => [
                'scan' => [
                    'paths' => [
                        __DIR__,
                    ],
                ],
            ],
        ];
    }
}
```

## Menyesuaikan Bahasa Lokalisasi Default

Jika sebelumnya Anda telah menggunakan komponen [hyperf/translation](https://github.com/hyperf/translation), periksa item konfigurasi `locale` di file `config/autoload/translation.php`. Jika nilainya `zh-CN`, ubah menjadi `zh_CN`. Di versi 1.1, kami menyatukan nilai konfigurasi ini.

## Menyesuaikan Dependensi composer.json

Karena Anda ingin meningkatkan ke komponen versi 1.1, dan proyek skeleton asli secara default bergantung pada komponen versi 1.0.x, kita perlu menyesuaikan batasan dependensi, mengubah semua dependensi komponen Hyperf asli dari `~1.0.0` menjadi `~1.1.0`. Setelah itu, jalankan `composer update` untuk meningkatkan dependensi ke versi 1.1.

Anda harus meningkatkan semua dependensi Hyperf ke versi 1.1 agar bisa digunakan, karena 1.1 menyesuaikan mekanisme ConfigProvider yang diadaptasi oleh komponen.

## Menyelesaikan Upgrade

Sejauh ini, upgrade ke 1.1 telah selesai. Namun, karena setiap file dasar Hyperf bisa ditulis ulang melalui DI, jika Anda telah menulis ulang beberapa file framework internal yang disesuaikan dalam upgrade ini, Anda masih perlu melakukan penyesuaian sesuai situasi aktual.

Jika Anda mengalami masalah selama atau setelah upgrade, silakan pergi ke [Github Issue](https://github.com/hyperf/hyperf/issues) untuk mengirimkan issue Anda, menjelaskan masalah yang Anda temui, dan kami akan membantu Anda menyelesaikannya sesegera mungkin.
