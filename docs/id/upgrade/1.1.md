# Panduan Upgrade 1.1

Versi 1.1 menambahkan banyak fungsi baru, tetapi beberapa perubahan juga
melibatkan penyesuaian skeleton dan struktur item konfigurasi. Jika Anda telah
membuat proyek aplikasi 1.0 berdasarkan proyek skeleton resmi untuk penggunaan
bisnis, Anda dapat menyesuaikan proyek skeleton Anda sesuai dengan poin-poin
berikut. Jika ini adalah proyek baru, ikuti dokumentasi untuk membuat proyek baru
menggunakan perintah `composer create-project hyperf/hyperf-skeleton` untuk
menggunakan struktur skeleton yang baru.

## Upgrade Swoole ke 4.4+

Versi 1.1 meningkatkan persyaratan versi Swoole minimum dari 4.3+ menjadi
4.4+. Ada beberapa detail penggunaan di antara kedua versi ini. Hyperf sudah
diadaptasikan pada versi sebelumnya, sehingga pengguna Hyperf dapat mengabaikan
perbedaan tersebut. Kami menaikkan persyaratan versi Swoole minimum terutama
untuk mengurangi beban historis kami, dan Swoole 4.4 sebagai versi LTS (Long
Term Support) Swoole juga berarti lebih stabil dan andal.

Hyperf akan melakukan deteksi versi Swoole saat startup, tetapi untuk
menyatukan dependensi versi Swoole di mana saja dengan lebih baik, kami
menyarankan Anda mengubah dependensi Swoole di `composer.json` menjadi
`"ext-swoole": ">=4.4"`.

## Menambahkan konstanta SWOOLE_HOOK_FLAGS

Tambahkan satu baris definisi konstanta ke entry file aplikasi `bin/hyperf.php`
dan entry file unit test `test/bootstrap.php` sebagai berikut:

```php
! defined('SWOOLE_HOOK_FLAGS') && define('SWOOLE_HOOK_FLAGS', SWOOLE_HOOK_ALL);
```

Referensi: [Referensi entry file](https://github.com/hyperf/hyperf-skeleton/blob/70062b7bbf29e23cda2f30680e02aa3b26ebd6f7/bin/hyperf.php#L11) [Referensi entry file unit test](https://github.com/hyperf/hyperf-skeleton/blob/70062b7bbf29e23cda2f30680e02aa3b26ebd6f7/test/bootstrap.php#L20)

## Memindahkan file config/dependencies.php dan menyesuaikan struktur file

Pindahkan `config/dependencies.php` → `config/autoload/dependencies.php`, dan
hapus level pertama `dependencies` dalam file konfigurasi, seperti berikut:

Struktur file 1.0:
```php
<?php
// config/dependencies.php document

return [
    'dependencies' => [
        FooInterface::class => Foo::class
    ],
];
```

Struktur file 1.1:
```php
<?php
// config/autoload/dependencies.php document

return [
    FooInterface::class => Foo::class
];
```

## Menyesuaikan isi file config/container.php

Karena versi 1.1 menyesuaikan lokasi dan struktur file `dependencies.php`, kita
juga perlu menyesuaikan file `config/container.php` agar dependency injection
container dapat berjalan dengan benar. File `config/container.php` menyediakan
cara penulisan yang lebih sederhana, `DefinitionSourceFactory` menggabungkan
banyak perilaku default, Anda hanya perlu mengganti konten dari file
`config/container.php` dengan konten berikut:

> Fitur cache annotation scan diaktifkan secara default. Anda dapat mengubah
> parameter pertama dari `DefinitionSourceFactory` untuk menonaktifkan fitur ini.

```php
<?php
/**
 * Initial a dependency injection container that implemented PSR-11 and return the container.
 */
declare(strict_types=1);

use Hyperf\Di\Container;
use Hyperf\Di\Definition\DefinitionSourceFactory;
use Hyperf\Utils\ApplicationContext;
use Psr\Container\ContainerInterface;

$container = new Container((new DefinitionSourceFactory(true))());
if (! $container instanceof ContainerInterface) {
    throw new RuntimeException('The dependency injection container is invalid.');
}
return ApplicationContext::setContainer($container);
```

## Menyesuaikan WebSocket controller

Karena versi 1.1 menyesuaikan batasan input (input constraints) dari `onMessage`
dan `onOpen`, Anda perlu mengubahnya secara manual menjadi
`Swoole\WebSocket\Server`, kode spesifiknya adalah sebagai berikut:

```php
<?php
declare(strict_types=1);

namespace App\Controller;

use Hyperf\Contract\OnMessageInterface;
use Hyperf\Contract\OnOpenInterface;
use Swoole\Http\Request;
use Swoole\Websocket\Frame;
use Swoole\WebSocket\Server as WebSocketServer;

class WebSocketController implements OnMessageInterface, OnOpenInterface
{
    public function onMessage(WebSocketServer $server, Frame $frame): void
    {
    }

    public function onOpen(WebSocketServer $server, Request $request): void
    {
    }
}
```

## Menyesuaikan ConfigProvider pada komponen kustom

Di versi 1.0, `scan.path` disesuaikan menjadi `annotations.scan.path` di versi
1.1. Anda perlu mengubah kelas ConfigProvider dari semua komponen kustom untuk
menyesuaikan dengan perubahan ini. Jika komponen kustom Anda tidak melibatkan
konfigurasi fitur annotation scan, penyesuaian ini dapat diabaikan, sebagai
berikut:

Struktur file ConfigProvider untuk 1.0:
```php
class ConfigProvider
{

    public function __invoke(): array
    {
        return [
            'scan' => [
                'paths' => [
                    __DIR__,
                ],
            ],
        ];
    }
}
```

Struktur file ConfigProvider untuk 1.1:
```php
class ConfigProvider
{

    public function __invoke(): array
    {
        return [
            'annotations' => [
                'scan' => [
                    'paths' => [
                        __DIR__,
                    ],
                ],
            ],
        ];
    }
}
```

## Menyesuaikan bahasa lokalisasi default

Jika Anda telah menggunakan komponen [hyperf/translation](https://github.com/hyperf/translation)
sebelumnya, maka Anda perlu memeriksa item konfigurasi `locale` di file
`config/autoload/translation.php`, misalnya jika nilainya adalah `zh-CN`, maka
perlu diubah menjadi `zh_CN`. Pada versi 1.1, kami menyatukan format nilai dari
konfigurasi ini.

## Menyesuaikan dependensi pada composer.json

Karena kita ingin melakukan upgrade ke versi komponen 1.1, dan proyek skeleton
asli bergantung pada versi komponen 1.0.x secara default, kita perlu melakukan
beberapa penyesuaian pada batasan dependensi (dependency constraints). Ubah semua
dependensi komponen asli Hyperf dari `~1.0.0` menjadi `~1.1.0`. Setelah
dimodifikasi, Anda perlu menjalankan `composer update` untuk memperbarui
dependensi ke versi 1.1.

Semua dependensi Hyperf harus di-upgrade ke versi 1.1 agar dapat digunakan,
karena versi 1.1 menyesuaikan mekanisme ConfigProvider untuk adaptasi
komponen.

## Menyelesaikan upgrade

Pada titik ini, upgrade ke versi 1.1 telah selesai. Namun, karena setiap file
underlying dari Hyperf dapat ditulis ulang (rewritten) melalui DI, jika Anda menulis
ulang beberapa file internal framework yang disesuaikan dalam upgrade ini, Anda
mungkin masih perlu melakukan beberapa penyesuaian sesuai dengan situasi aktual
Anda.

Jika Anda menemui masalah apa pun selama atau setelah proses upgrade, silakan
kunjungi [Github Issue](https://github.com/hyperf/hyperf/issues) untuk mengirimkan
laporan masalah Anda, jelaskan kendala yang Anda alami, dan kami akan mencoba
membantu menyelesaikannya sesegera mungkin.
