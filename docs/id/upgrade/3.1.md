# 3.1 Upgrade Guide

- Versi 3.1 terutama mengubah versi minimum `PHP` menjadi `8.1` dan versi minimum `Swoole` menjadi `5.0`.
- Memperkenalkan framework testing `Pest`.
- Menambahkan `hyperf/helper`, dan fungsi pembantu (helper functions) asli `tanpa namespace` di `Hyperf\Utils` telah dipindahkan ke `hyperf/helper`.
- Mengubah cara pemuatan file konfigurasi multi-level `hyperf/config`, mendukung sintaks `.`, seperti `config('a.c')`.

## Perubahan pada Utils Package

Fungsi pembantu di utils package asli tidak memiliki namespace, yang berpotensi bertentangan dengan fungsi di package composer lain. Fungsi-fungsi tersebut telah dihapus dan diganti di 3.1.

Ada dua cara untuk menangani ini:

1. Jika Anda yakin tidak ada package yang dimuat yang akan menyebabkan konflik fungsi, Anda bisa langsung menginstal `hyperf/helper`. Package `helper` menyediakan fungsi pembantu tanpa namespace, yang konsisten dengan yang ada di `utils` asli.
2. Tambahkan namespace di tempat fungsi pembantu awalnya dipanggil. Anda dapat menggunakan `rector` untuk refactoring. Lihat [dokumentasi refactoring](https://github.com/orgs/hyperf/discussions/5635).

## Perubahan Cara Pemuatan File Konfigurasi

Di 3.0, konfigurasi menggunakan `nama file konfigurasi` sebagai key. Di 3.1, pemuatan file konfigurasi multi-level `hyperf/config` menggunakan `direktori relatif.nama file konfigurasi` sebagai key.

Ini mendukung sintaks `.`, seperti `config('a.c')`.

Perhatikan penanganan dua situasi berikut di proyek yang ada:

- Di proyek 3.0, jika Anda membuat subdirektori di `config/autoload`, seperti `config/autoload/sub/a.php`, `config('a')` yang asli perlu diubah menjadi `config('sub.a')`.
- Jika nama file konfigurasi di proyek 3.0 mengandung `.`, seperti `config/autoload/a.b.php`, maka akan mengembalikan struktur berikut saat menggunakan `config('a')`. Jika Anda juga memiliki file konfigurasi `config/autoload/a.php`, Anda juga akan mendapatkan hasil setelah penggabungan item konfigurasi.

```php
return [
    'a.php 中的配置项' => 'a.php 中的配置值',
    'b' => [
        'a.b.php 中的配置项' => 'a.b.php 中的配置值',
    ]
];
```

## Perubahan Lainnya

Mulai dari v3.1, komponen `hyperf/metric` tidak lagi menginstal dependensi `prometheus` secara default. Oleh karena itu, pengembang yang perlu menggunakan engine `prometheus` harus menjalankan perintah berikut untuk memuat dependensi:

```bash
composer require promphp/prometheus_client_php
```

Fungsionalitas `withHeaders` dari `Request` dan `Response` telah berubah.

Versi sebelum v3.1 akan menyimpan data asli, mirip dengan `array_merge`. Setelah v3.1, ini langsung mengganti semua `headers`.

```php
<?php
// Sebelum v3.1
$request->withHeader('foo', 1)->withHeader('bar', 2); // ['foo' => [1], 'bar' => [2]]
$request->withHeader('foo', 1)->withHeader('foo', 2); // ['foo' => [2]]
$request->withAddedHeader('foo', 1)->withAddedHeader('foo', 2); // ['foo' => [1, 2]]

$request->withHeaders(['foo' => 1])->withHeaders(['bar' => 2]); // ['foo' => [1], 'bar' => [2]]

// Setelah v3.1
$request->withHeader('foo', 1)->withHeader('bar', 2); // ['foo' => [1], 'bar' => [2]]
$request->withHeader('foo', 1)->withHeader('foo', 2); // ['foo' => [2]]
$request->withAddedHeader('foo', 1)->withAddedHeader('foo', 2); // ['foo' => [1, 2]]

$request->withHeaders(['foo' => 1])->withHeaders(['bar' => 2]); // ['bar' => [2]]

$request->setHeader('foo', 1)->setHeader('bar', 2); // ['foo' => [1], 'bar' => [2]]
$request->setHeader('foo', 1)->setHeader('foo', 2); // ['foo' => [2]]
$request->addHeader('foo', 1)->addHeader('foo', 2); // ['foo' => [1, 2]]

$request->setHeaders(['foo' => 1])->setHeaders(['bar' => 2]); // ['bar' => [2]]
```

3. Return value dari method `consumeMessage` pada consumer `AMQP` disesuaikan menjadi `Result` enum.

```php
<?php
// Sebelum v3.1
public function consumeMessage($data, AMQPMessage $message): string
{
    return Result::ACK;
}
// Setelah v3.1
public function consumeMessage($data, AMQPMessage $message): Result
{
    return Result::ACK;
}
```
