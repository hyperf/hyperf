# Panduan Upgrade 3.1

- Versi 3.1 terutama mengubah versi minimum `PHP` menjadi `8.1` dan versi minimum `Swoole` menjadi `5.0`.
- Memperkenalkan testing framework `Pest`
- Menambahkan `hyperf/helper`, memigrasikan `helper function tanpa namespace` dari `Hyperf\Utils` ke `hyperf/helper`.
- Mengubah cara `hyperf/config` memuat file konfigurasi multi-level untuk mendukung sintaks `.`, contohnya `config('a.c')`

## Perubahan Paket Utils

Helper function pada paket utils yang asli tidak memiliki namespace, dan
mungkin berkonflik dengan fungsi pada paket composer lainnya, sehingga dihapus
dan diganti di 3.1.

Ada dua cara untuk menangani hal ini:

- Jika tidak ada paket lain yang dimasukkan ke dalam aplikasi yang menyebabkan
  konflik fungsi, Anda dapat menginstal `hyperf/helper` secara langsung, dan
  paket `helper` tersebut menyediakan helper function tanpa namespace, yang
  sama seperti paket `utils` yang asli.
- Namespace diperkenalkan di tempat helper function dipanggil, dan dapat
  direfaktor menggunakan `rector` [dokumentasi refactoring di sini 🔎](https://github.com/orgs/hyperf/discussions/5635)

## Perubahan Pemuatan File Konfigurasi

Pada versi 3.0, konfigurasi ada dengan `nama file konfigurasi` sebagai key,
sedangkan pada versi 3.1, file konfigurasi multi-tier pada `hyperf/config`
dimuat dengan `direktori relatif.Nama file konfigurasi` sebagai key.

Dukungan untuk sintaks `.`, seperti `config('a.c')`.

Dalam hal ini, penting untuk diperhatikan bagaimana kedua kasus berikut
ditangani pada proyek asli:

- Proyek versi 3.0 telah membuat subdirektori di dalam `config/autoload`,
  contohnya `config/autoload/sub/a.php`, maka `config('a')` yang asli perlu
  diubah menjadi `config('sub.a')`.
- Nama file konfigurasi pada proyek versi 3.0 yang mengandung `.`, seperti
  `config/autoload/a.b.php`, akan mengembalikan struktur berikut saat
  menggunakan `config('a')`. Jika Anda juga memiliki file konfigurasi
  `config/autoload/a.php`, Anda juga akan mendapatkan hasil setelah item
  konfigurasi digabungkan.
```php
return [
    'a.php config key' => 'a.php config value',
    'b' => [
        'a.b.php config key' => 'a.b.php config value',
    ]
];
```

## Perubahan Lainnya

1. Komponen `hyperf/metric` tidak lagi menginstal dependensi `prometheus`
   secara default, sehingga developer yang perlu menggunakan engine
   `prometheus` harus menjalankan perintah berikut untuk memuat dependensi
   tersebut:

```bash
composer require promphp/prometheus_client_php
```

2. Perubahan pada `withHeaders` untuk `Request` dan `Response`

Sebelum v3.1, data asli akan disimpan, serupa dengan `array_merge`. Setelah
versi v3.1, semua `headers` akan langsung diganti.

```php
<?php
// before v3.1 
$request->withHeader('foo', 1)->withHeader('bar', 2); // ['foo' => [1], 'bar' => [2]]
$request->withHeader('foo', 1)->withHeader('foo', 2); // ['foo' => [2]]
$request->withAddedHeader('foo', 1)->withAddedHeader('foo', 2); // ['foo' => [1, 2]]

$request->withHeaders(['foo' => 1])->withHeaders(['bar' => 2]); // ['foo' => [1], 'bar' => [2]]

// after v3.1
$request->withHeader('foo', 1)->withHeader('bar', 2); // ['foo' => [1], 'bar' => [2]]
$request->withHeader('foo', 1)->withHeader('foo', 2); // ['foo' => [2]]
$request->withAddedHeader('foo', 1)->withAddedHeader('foo', 2); // ['foo' => [1, 2]]

$request->withHeaders(['foo' => 1])->withHeaders(['bar' => 2]); // ['bar' => [2]]

$request->setHeader('foo', 1)->setHeader('bar', 2); // ['foo' => [1], 'bar' => [2]]
$request->setHeader('foo', 1)->setHeader('foo', 2); // ['foo' => [2]]
$request->addHeader('foo', 1)->addHeader('foo', 2); // ['foo' => [1, 2]]

$request->setHeaders(['foo' => 1])->setHeaders(['bar' => 2]); // ['bar' => [2]]
```

3. Nilai kembalian method `consumeMessage` pada consumer `AMQP` disesuaikan
   menjadi enum Result

```php
<?php
// before v3.1
public function consumeMessage($data, AMQPMessage $message): string
{
    return Result::ACK;
}
// after v3.1
public function consumeMessage($data, AMQPMessage $message): Result
{
    return Result::ACK;
}
```
