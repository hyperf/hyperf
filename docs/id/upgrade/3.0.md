# Panduan Upgrade 3.0

- Versi 3.0 terutama mengubah persyaratan `PHP`, versi minimumnya adalah `8.0`.
- Framework menghapus `Doctrine Annotations` dan menggunakan `PHP8 Attributes`
  sebagai gantinya.
- Framework menambahkan banyak batasan tipe untuk member variable.

## Mengonversi semua anotasi

**Catatan: Langkah ini hanya dapat dilakukan di bawah versi 2.2**

Skrip berikut akan mengonversi semua `Doctrine Annotations` menjadi `PHP8
Attributes`.

```shell
composer require hyperf/code-generator
php bin/hyperf.php code:generate -D app
```

## Mengubah semua versi komponen Hyperf

Cukup ubah `hyperf/*` di `composer.json` menjadi `3.0.*`.

> `hyperf/engine` tidak mengikuti nomor versi framework, jadi tidak perlu
> mengubahnya, cukup pastikan versinya adalah `^2.1.0`.

Setelah itu, Anda hanya perlu menjalankan `composer update -o`, dan proses
upgrade akan selesai secara normal.

## Upgrade model database

Karena kelas dasar model (model base class) menambahkan dukungan tipe untuk
member variable, Anda perlu menggunakan skrip berikut untuk meningkatkannya ke
versi baru.

```shell
composer require hyperf/code-generator
php vendor/bin/regenerate-models.php $PWD/app/Model
```

## Logger

Versi `monolog/monolog` 3.x menggunakan fitur baru PHP 8.1, sehingga beberapa
kelas perlu dimodifikasi secara khusus.

Ubah `array $record` menjadi `array|LogRecord $record` agar kompatibel dengan
versi 3.x, contoh kodenya adalah sebagai berikut:

```php
<?php

declare(strict_types=1);

namespace App\Kernel\Log;

use Hyperf\Context\Context;
use Hyperf\Coroutine\Coroutine;
use Monolog\LogRecord;
use Monolog\Processor\ProcessorInterface;

class AppendRequestIdProcessor implements ProcessorInterface
{
    public const REQUEST_ID = 'log.request.id';

    public function __invoke(array|LogRecord $record)
    {
        $record['extra']['request_id'] = Context::getOrSet(self::REQUEST_ID, uniqid());
        $record['extra']['coroutine_id'] = Coroutine::id();
        return $record;
    }
}
```

## Command

Setelah versi `3.0`, command line mengaktifkan event listener secara default.
Oleh karena itu, ketika sebuah listener mendengarkan event `Command` dan
menjalankan `AMQP` atau logika multiplexing lainnya, proses tersebut tidak akan
dapat keluar (exit).

Solusi:

- Metode 1:

Saat menjalankan command, tambahkan opsi `--disable-event-dispatcher`.

- Metode 2:

Tambahkan listener berikut:

```php
<?php

declare(strict_types=1);

namespace App\Listener;

use Hyperf\Command\Event\AfterExecute;
use Hyperf\Coordinator\Constants;
use Hyperf\Coordinator\CoordinatorManager;
use Hyperf\Event\Annotation\Listener;
use Hyperf\Event\Contract\ListenerInterface;

#[Listener]
class ResumeExitCoordinatorListener implements ListenerInterface
{
    public function listen(): array
    {
        return [
            AfterExecute::class,
        ];
    }

    public function process(object $event): void
    {
        CoordinatorManager::until(Constants::WORKER_EXIT)->resume();
    }
}
```

## Menjalankan server

Selanjutnya, Anda hanya perlu menjalankan server, dan Anda akan dapat melihat
kode yang tidak kompatibel lalu memperbaikinya satu per satu.

- Menambahkan tipe untuk member variable pada AMQP Consumer dan Producer.
- Metode `process` pada Listener menambahkan tipe kembalian `void`.
- Parameter `$timeout` pada annotation `#[CircuitBreaker]` diubah menjadi
  `$options.timeout`.

## GRPC

Framework mengubah `Http status` yang dikembalikan oleh `GRPC Server` menjadi
200 sesuai dengan spesifikasi `GRPC`, dan `GRPC Server` mengembalikan `status
code` yang sesuai. Silakan tingkatkan ke versi `3.x` pada client/peer, jika tidak
sisi peer tidak akan dapat mengurai error secara normal ketika request tidak
normal.
