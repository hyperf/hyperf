# 3.0 Upgrade Guide

- Versi 3.0 terutama mengubah versi minimum `PHP` menjadi `8.0`.
- Framework telah menghapus `Doctrine Annotations` dan menggantinya dengan `PHP8 Attributes`.
- Framework telah menambahkan banyak batasan tipe member variable.

## Mengonversi Semua Annotations ke Native Attributes

**Catatan: Langkah ini hanya dapat dijalankan di bawah versi 2.2.**

Skrip berikut akan mengonversi semua `Doctrine Annotations` menjadi `PHP8 Attributes`.

```shell
composer require hyperf/code-generator
php bin/hyperf.php code:generate -D app
```

## Memodifikasi Versi Komponen Hyperf

Ubah langsung semua `hyperf/*` di `composer.json` menjadi `3.0.*`.

Pastikan versinya `^2.1.0`. Jika menggunakan Hyperf di bawah v3.0, gunakan engine component versi `^1.5`.

Setelah itu, jalankan saja `composer update -o` untuk menyelesaikan upgrade secara normal.

## Meng-upgrade Kelas Model Database

Karena kelas dasar model telah menambahkan dukungan tipe untuk member variables, Anda perlu menggunakan skrip berikut untuk meningkatkannya ke versi baru.

```shell
composer require hyperf/code-generator
php vendor/bin/regenerate-models.php $PWD/app/Model
```

## Logger

`monolog/monolog` versi 3.x menggunakan fitur baru PHP8.1, sehingga diperlukan modifikasi khusus untuk beberapa kelas.

Ubah `array $record` menjadi `array|LogRecord $record` agar kompatibel dengan versi 3.x. Contoh kode adalah sebagai berikut:

```php
<?php

declare(strict_types=1);

namespace App\Kernel\Log;

use Hyperf\Context\Context;
use Hyperf\Coroutine\Coroutine;
use Monolog\LogRecord;
use Monolog\Processor\ProcessorInterface;

class AppendRequestIdProcessor implements ProcessorInterface
{
    public const REQUEST_ID = 'log.request.id';

    public function __invoke(array|LogRecord $record)
    {
        $record['extra']['request_id'] = Context::getOrSet(self::REQUEST_ID, uniqid());
        $record['extra']['coroutine_id'] = Coroutine::id();
        return $record;
    }
}
```

## Command

Setelah versi `3.0`, command line mengaktifkan event listeners secara default. Oleh karena itu, jika ada listener yang mendengarkan event `Command`, dan menjalankan logika multiplexing seperti `AMQP`, proses akan gagal keluar.

Solusi:

- Metode 1:

Saat menjalankan command, tambahkan opsi `--disable-event-dispatcher`.

- Metode 2:

Tambahkan listener:

```php
<?php

declare(strict_types=1);

namespace App\Listener;

use Hyperf\Command\Event\AfterExecute;
use Hyperf\Coordinator\Constants;
use Hyperf\Coordinator\CoordinatorManager;
use Hyperf\Event\Annotation\Listener;
use Hyperf\Event\Contract\ListenerInterface;

#[Listener]
class ResumeExitCoordinatorListener implements ListenerInterface
{
    public function listen(): array
    {
        return [
            AfterExecute::class,
        ];
    }

    public function process(object $event): void
    {
        CoordinatorManager::until(Constants::WORKER_EXIT)->resume();
    }
}
```

## Mengganti Hyperf\Utils\Context

- Karena namespace dari `Context` telah berubah di 3.0, Anda harus mengganti secara global `Hyperf\Utils\Context` => `Hyperf\Context\Context`.

## Memulai Service

Selanjutnya, jalankan saja service, dan Anda dapat melihat di mana yang belum diadaptasi, lalu memodifikasinya satu per satu.

- Member variables pada AMQP Consumer dan Producer telah ditambahkan tipe.
- Method `process` pada Listener telah menambahkan return type `void`.
- Annotation `CircuitBreaker` mengubah parameter `$timeout` menjadi `$options.timeout`.
- Member variables pada Async Queue telah ditambahkan tipe.
- Async Queue, field `event->message` diubah menjadi tipe non-public, dan Anda perlu menggunakan `event->getMessage()` untuk membacanya.

## gRPC

Framework mengubah `Http status` yang dikembalikan oleh `gRPC Server` menjadi tetap 200 sesuai spesifikasi `gRPC`. `gRPC Server` mengembalikan `status code` yang sesuai. Jika Anda menggunakan `gRPC` sebelum update ini, pastikan untuk meng-upgrade layanan terkait ke versi `3.x`, jika tidak, pihak lawan tidak akan bisa mengurai error secara normal saat terjadi exception pada permintaan.
