# Panduan Upgrade 2.1

- Versi 2.1 terutama menambahkan driver dasar
  [hyperf/engine](https://github.com/hyperf/engine), memungkinkan framework
  `Hyperf` untuk berjalan di atas `Swoole` atau `Swow`.
- Versi PHP minimum berubah dari 7.2 menjadi 7.3

> Swow untuk sementara adalah versi preview, harap gunakan dengan hati-hati.

## Tambahkan konfigurasi optimize-autoloader

Karena `Hyperf` perlu menggunakan `class_map` yang dihasilkan oleh `composer`
sejak `2.0`, yang mengharuskan pengguna untuk menggunakan `-o` untuk optimasi
setiap kali mereka memperbarui dependensi, tetapi banyak pengguna tidak memiliki
kebiasaan ini.
Oleh karena itu, kami menyarankan untuk menambahkan konfigurasi yang sesuai ke
`composer.json` untuk memenuhi kebutuhan ini.

```json
{
    "config": {
        "optimize-autoloader": true,
        "sort-packages": true
    }
}
```

## Ubah SwooleEvent

Class `Hyperf\Server\SwooleEvent` telah diganti namanya menjadi
`Hyperf\Server\Event` pada `2.1`, sehingga kita perlu mengubah kode yang sesuai
dalam konfigurasi `server.php`.

> SwooleEvent akan dihapus secara resmi di 3.0, silakan ubah menjadi Event
> sesegera mungkin

```php
<?php

declare(strict_types=1);

use Hyperf\Server\Event;
use Hyperf\Server\Server;

return [
    'mode' => SWOOLE_BASE,
    'servers' => [
        [
            'name' => 'http',
            'type' => Server::SERVER_HTTP,
            'host' => '0.0.0.0',
            'port' => 9501,
            'sock_type' => SWOOLE_SOCK_TCP,
            'callbacks' => [
                Event::ON_REQUEST => [Hyperf\HttpServer\Server::class, 'onRequest'],
            ],
        ],
    ],
    'callbacks' => [
        Event::ON_BEFORE_START => [Hyperf\Framework\Bootstrap\ServerStartCallback::class, 'beforeStart'],
        Event::ON_WORKER_START => [Hyperf\Framework\Bootstrap\WorkerStartCallback::class, 'onWorkerStart'],
        Event::ON_PIPE_MESSAGE => [Hyperf\Framework\Bootstrap\PipeMessageCallback::class, 'onPipeMessage'],
        Event::ON_WORKER_EXIT => [Hyperf\Framework\Bootstrap\WorkerExitCallback::class, 'onWorkerExit'],
    ],
];
```

## Paginator

Karena komponen `hyperf/paginator` telah dihapus dari dependensi
`hyperf/database`. Oleh karena itu, developer yang menggunakan paginator di
database juga perlu memasukkan komponen `hyperf/paginator`.

## Ubah versi DBAL

Jika Anda menggunakan komponen `doctrine/dbal`, Anda perlu melakukan upgrade ke
versi `^3.0`.

## Hapus komponen doctrine/common

Komponen `doctrine/common` memiliki konflik dependensi dengan `hyperf/utils`.
Jadi komponen ini perlu dihapus dari `composer.json`.

```bash
# remove component
composer remove doctrine/common
# update
composer update "hyperf/*" -o
```

## Tindakan Pencegahan

- Cobalah untuk tidak mengubah engine dari proyek lama ke Swow, jika Anda ingin
  menggunakan Swow, silakan mencobanya di proyek baru. Karena Swow bukan
  pengganti Swoole, tidak semua skenario Swoole dapat menemukan alternatif yang
  sesuai di Swow.
