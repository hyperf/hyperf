# 2.1 Upgrade Guide

- Versi 2.1 terutama menambahkan driver dasar [hyperf/engine](https://github.com/hyperf/engine), yang memungkinkan framework `Hyperf` berjalan di atas `Swoole` atau `Swow`.
- Versi minimum PHP diubah dari 7.2 menjadi 7.3.

> Swow saat ini adalah versi pratinjau, gunakan dengan hati-hati.

## Menambahkan Konfigurasi optimize-autoloader

Karena `Hyperf` sudah membutuhkan `class_map` yang dihasilkan oleh `composer` sejak `2.0`, ini mengharuskan pengguna menggunakan `-o` untuk optimasi setiap kali dependensi diperbarui, tapi banyak pengguna tidak memiliki kebiasaan ini.
Karena itu, kami sarankan menambahkan konfigurasi yang sesuai di `composer.json`.

```json
{
    "config": {
        "optimize-autoloader": true,
        "sort-packages": true
    }
}
```

## Memodifikasi SwooleEvent

Kelas `Hyperf\Server\SwooleEvent` telah diubah namanya menjadi `Hyperf\Server\Event` di `2.1`, jadi kita perlu memodifikasi kode yang sesuai di konfigurasi `server.php`.

> SwooleEvent akan resmi dihapus di 3.0, segera ubah ke Event.

```php
<?php

declare(strict_types=1);

use Hyperf\Server\Event;
use Hyperf\Server\Server;

return [
    'mode' => SWOOLE_BASE,
    'servers' => [
        [
            'name' => 'http',
            'type' => Server::SERVER_HTTP,
            'host' => '0.0.0.0',
            'port' => 9501,
            'sock_type' => SWOOLE_SOCK_TCP,
            'callbacks' => [
                Event::ON_REQUEST => [Hyperf\HttpServer\Server::class, 'onRequest'],
            ],
        ],
    ],
    'callbacks' => [
        Event::ON_BEFORE_START => [Hyperf\Framework\Bootstrap\ServerStartCallback::class, 'beforeStart'],
        Event::ON_WORKER_START => [Hyperf\Framework\Bootstrap\WorkerStartCallback::class, 'onWorkerStart'],
        Event::ON_PIPE_MESSAGE => [Hyperf\Framework\Bootstrap\PipeMessageCallback::class, 'onPipeMessage'],
        Event::ON_WORKER_EXIT => [Hyperf\Framework\Bootstrap\WorkerExitCallback::class, 'onWorkerExit'],
    ],
];
```

## Paginator

Karena komponen `hyperf/paginator` telah dihapus dari dependensi `hyperf/database`. Pengguna yang menggunakan paginator di database perlu menambahkan komponen `hyperf/paginator` secara terpisah.

## Memodifikasi Versi DBAL

Jika Anda menggunakan komponen `doctrine/dbal`, Anda perlu meningkatkan ke versi `^3.0`.

## Menghapus Komponen doctrine/common

Ada konflik dependensi antara `doctrine/common` dan `hyperf/utils`. Oleh karena itu, hapus komponen ini dari `composer.json`.

```bash
# Hapus komponen
composer remove doctrine/common
# Perbarui
composer update "hyperf/*" -o
```

## Tindakan Pencegahan

- Usahakan untuk tidak mengubah engine proyek lama ke Swow. Jika ingin menggunakan Swow, cobalah di proyek baru. Karena Swow bukan pengganti Swoole, tidak semua skenario Swoole memiliki solusi alternatif yang sesuai di Swow.
