# 2.2 Upgrade Guide

Versi 2.2 terutama menambahkan dukungan untuk `PHP 8` dan native Annotations.

## Memodifikasi Versi Komponen Hyperf

Ubah langsung semua `hyperf/*` di `composer.json` menjadi `2.2.*`.

> `hyperf/engine` tidak mengikuti nomor versi framework, jadi tidak perlu dimodifikasi.

Selain itu, kita bisa menjalankan `composer require "hyperf/ide-helper:2.2.*" --dev` untuk menginstal `hyperf/ide-helper`. Komponen ini membantu memberikan petunjuk untuk parameter yang bisa diatur di Annotations saat menggunakan native Annotations.

Setelah itu, jalankan saja `composer update -o` untuk menyelesaikan upgrade secara normal.

## Memodifikasi Skrip Unit Test

Tambahkan opsi `--prepend test/bootstrap.php`:

```json
{
    "scripts": {
        "test": "co-phpunit --prepend test/bootstrap.php -c phpunit.xml --colors=always"
    }
}
```

## Menginstal Ekstensi pcntl

Versi baru Annotation scanning menggunakan ekstensi `pcntl`, jadi pastikan `PHP` Anda telah menginstal ekstensi ini.

```shell
php --ri pcntl

pcntl
pcntl support => enabled
```

> Saat `grpc` diaktifkan, Anda perlu menambahkan `grpc.enable_fork_support= 1;` ke `php.ini` untuk mendukung pengaktifan child processes.

## AMQP

> Pengguna yang menggunakan AMQP, harap diperhatikan; yang tidak menggunakannya dapat mengabaikan bagian ini.

Karena komponen `AMQP` telah sepenuhnya di-upgrade untuk mendukung multiplexing, ada beberapa perubahan konfigurasi. Silakan sesuaikan dengan konfigurasi terbaru di bawah ini.

```php
<?php

return [
    'default' => [
        'host' => env('AMQP_HOST', 'localhost'),
        'port' => (int) env('AMQP_PORT', 5672),
        'user' => env('AMQP_USER', 'guest'),
        'password' => env('AMQP_PASSWORD', 'guest'),
        'vhost' => env('AMQP_VHOST', '/'),
        'concurrent' => [
            'limit' => 1,
        ],
        'pool' => [
            // Jumlah koneksi yang dibuka secara bersamaan
            // Karena versi baru koneksi mendukung multiplexing, Anda dapat mencapai konkurensi tinggi dengan sangat sedikit koneksi
            'connections' => 2,
        ],
        'params' => [
            'insist' => false,
            'login_method' => 'AMQPLAIN',
            'login_response' => null,
            'locale' => 'en_US',
            'connection_timeout' => 3,
            'read_write_timeout' => 6,
            'context' => null,
            'keepalive' => true,
            'heartbeat' => 3,
            'channel_rpc_timeout' => 0.0,
            'close_on_destruct' => false,
            // Jumlah maksimum Channel idle dalam multiplexing, melebihi jumlah ini akan menutup channel terbatas yang berlebih
            'max_idle_channels' => 10,
        ],
    ],
];
```

## Configuration Center

> Pengguna yang menggunakan Configuration Center, harap diperhatikan; yang tidak menggunakannya dapat mengabaikan bagian ini.

Configuration Center telah direfaktor ulang sepenuhnya di versi ini. Pastikan untuk membaca ulang dokumentasi yang sesuai dengan saksama.

Anda perlu memperkenalkan komponen `hyperf/config-center`. Perintahnya adalah sebagai berikut:

```shell
composer require "hyperf/config-center:~2.2.0"
```

Dan perkenalkan komponen dependensi driver yang sesuai berdasarkan driver yang digunakan. Misalnya, jika Anda menggunakan `Apollo`, Anda perlu memperkenalkan komponen `hyperf/config-apollo`. Driver lain serupa.

Pada saat yang sama, semua informasi konfigurasi yang terkait dengan configuration center telah dikumpulkan di `config/autoload/config_center.php`. Silakan konfigurasikan sesuai dengan struktur konfigurasi baru. Jika file ini tidak ada, Anda dapat membuatnya dengan menjalankan perintah `php bin/hyperf.php vendor:publish hyperf/config-center`.

## Service Center

Pengguna yang menggunakan komponen `hyperf/service-governance`: karena adapter `consul` telah dipisahkan dari komponen ini, Anda perlu menambahkan komponen `hyperf/service-governance-consul` di versi baru. Perintahnya adalah sebagai berikut:

```shell
composer require "hyperf/service-governance-consul:~2.2.0"
```

Pengguna yang menggunakan `nacos` sebagai driver service center perlu memperkenalkan komponen `hyperf/service-governance-nacos`. Perintahnya adalah sebagai berikut:

```shell
composer require "hyperf/service-governance-nacos:~2.2.0"
```

## php-cs-fixer

Jika Anda tidak perlu meningkatkan `php-cs-fixer` ke versi `3.0`, Anda dapat mengabaikan bagian ini.

1. Ubah nomor versi

```
"friendsofphp/php-cs-fixer": "^3.0"
```

2. Ubah nama file .php_cs

Ubah nama menjadi `.php-cs-fixer.php` dan modifikasi kode yang sesuai sesuai dengan catatan perubahan berikut:

```diff
- return PhpCsFixer\Config::create()
+ return (new PhpCsFixer\Config())

- 'commentType' => 'PHPDoc',
+ 'comment_type' => 'PHPDoc',
```

## Kompatibilitas DI Menurun di PHP 7.3

> Pengguna yang menggunakan versi PHP yang lebih tinggi dapat mengabaikan bagian ini.

Di versi `2.0` - `2.1`, untuk mengimplementasikan `AOP` pada objek yang tidak dikelola oleh `DI` (seperti objek yang dibuat dengan keyword `new`), implementasi dasar menggunakan komponen `BetterReflection`. Selain memberikan pengalaman pemrograman baru, ini juga membawa beberapa masalah yang sulit diatasi, seperti:

- Startup proyek sangat lambat ketika tidak ada scan cache
- `Inject` dan `Value` tidak berfungsi di skenario khusus
- `BetterReflection` belum mendukung PHP 8 (pada saat rilis v2.2)

Di versi baru, penggunaan `BetterReflection` tidak lagi direkomendasikan, dan `sub-process scanning` digunakan untuk menyelesaikan masalah-masalah ini. Namun, ada beberapa ketidakcocokan di versi `PHP` yang lebih rendah:

Setelah menggunakan `PHP 7.3` untuk memulai aplikasi, Anda mungkin mengalami error yang mirip dengan berikut:

```bash
PHP Fatal error:  Interface 'Hyperf\Signal\SignalHandlerInterface' not found in vendor/hyperf/process/src/Handler/ProcessStopHandler.php on line 17

PHP Fatal error:  Interface 'Symfony\Component\Serializer\SerializerInterface' not found in vendor/hyperf/utils/src/Serializer/Serializer.php on line 46
```

Masalah ini terjadi karena di `PHP 7.3`, ketika mendapatkan reflection melalui `sub-process scanning`, jika sebuah kelas mengimplementasikan `Interface` yang tidak ada, maka akan menyebabkan exception `Interface not found` dilemparkan, sementara versi `PHP` yang lebih tinggi tidak akan mengalami hal ini.

Solusinya adalah dengan membuat `Interface` yang sesuai dan memperkenalkannya secara normal. Solusi untuk error yang disebutkan di atas adalah dengan menginstal komponen dependensi yang sesuai.

> Tentu saja, yang terbaik adalah meningkatkan ke versi 7.4 atau 8.0

```bash
composer require hyperf/signal

composer require symfony/serializer
```

## File System

> Pengguna yang menggunakan File System, harap diperhatikan; yang tidak menggunakannya dapat mengabaikan bagian ini.

Mulai dari versi v2.2, Hyperf mendukung penggunaan komponen `League\Flysystem` versi `v1.0` dan `v2.0` secara bersamaan.

Jika Anda menggunakan `League\Flysystem` versi `v2.0`, adapter yang sesuai juga perlu disesuaikan menurut daftar berikut. Untuk dokumentasi spesifik, silakan lihat bab `File System`.

- Alibaba Cloud OSS adapter

```shell
composer require hyperf/flysystem-oss
```

- S3 adapter

```shell
composer require "league/flysystem-aws-s3-v3:^2.0"
composer require hyperf/guzzle
```

- Qiniu adapter

```shell
composer require "overtrue/flysystem-qiniu:^2.0"
```

- Memory adapter

```shell
composer require "league/flysystem-memory:^2.0"
```

- Tencent Cloud COS adapter

> Konfigurasi dasar adapter ini sedikit berbeda dari versi `v2.0` komponen `overtrue/flysystem-cos`. Silakan lakukan modifikasi yang sesuai sesuai dengan konfigurasi versi terbaru.

```shell
composer require "overtrue/flysystem-cos:^4.0"
```
