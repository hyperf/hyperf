# Panduan Peningkatan 2.2

Versi 2.2 terutama menambahkan adaptasi `PHP 8` dan mendukung native
annotations.

## Ubah versi komponen Hyperf

Cukup ubah `hyperf/*` di `composer.json` menjadi `2.2.*`.

> `hyperf/engine` tidak mengikuti nomor versi framework, sehingga tidak
> memerlukan modifikasi.

Selain itu, Anda dapat menjalankan `composer require "hyperf/ide-helper:2.2.*"
--dev` untuk menginstal `hyperf/ide-helper`. Komponen ini dapat membantu
memberikan petunjuk parameter yang dapat diatur saat menggunakan native
annotations.

Setelah itu, Anda hanya perlu menjalankan `composer update -o`, dan proses
peningkatan dapat diselesaikan dengan normal.

## Ubah skrip pengujian tunggal

Menambahkan opsi `--prepend test/bootstrap.php`

```json
{
    "scripts": {
        "test": "co-phpunit --prepend test/bootstrap.php -c phpunit.xml --colors=always"
    }
}
```

## Instal ekstensi pcntl

Versi baru dari pemindaian anotasi (annotation scanning) menggunakan ekstensi
`pcntl`, jadi pastikan `PHP` Anda telah menginstal ekstensi ini terlebih dahulu.

```shell
php --ri pcntl

pcntl

pcntl support => enabled
```

> Saat mengaktifkan `grpc`, Anda perlu menambahkan `grpc.enable_fork_support = 1;`
> ke dalam `php.ini` untuk mendukung pembukaan child processes.

## AMQP

> Perhatian bagi pengguna yang menggunakan AMQP, bagian ini dapat diabaikan
> jika Anda tidak menggunakannya.

Karena komponen `AMQP` telah ditingkatkan secara menyeluruh dan mendukung
multiplexing, terdapat beberapa perubahan dalam konfigurasi. Silakan ikuti
konfigurasi terbaru di bawah ini dan sesuaikan seperlunya.

```php
<?php

return [
    'default' => [
        'host' => env('AMQP_HOST', 'localhost'),
        'port' => (int) env('AMQP_PORT', 5672),
        'user' => env('AMQP_USER', 'guest'),
        'password' => env('AMQP_PASSWORD', 'guest'),
        'vhost' => env('AMQP_VHOST', '/'),
        'concurrent' => [
            'limit' => 1,
        ],
        'pool' => [
            // Number of connections opened at the same time
            // Because the new version of the connection supports multiplexing, it can achieve high concurrency with a very small number of connections
            'connections' => 2,
        ],
        'params' => [
            'insist' => false,
            'login_method' => 'AMQPLAIN',
            'login_response' => null,
            'locale' => 'en_US',
            'connection_timeout' => 3,
            'read_write_timeout' => 6,
            'context' => null,
            'keepalive' => true,
            'heartbeat' => 3,
            'channel_rpc_timeout' => 0.0,
            'close_on_destruct' => false,
            // The maximum value of idle channels in multiplexing. After this number is exceeded, the redundant limit channels will be closed.
            'max_idle_channels' => 10,
        ],
    ],
];

```

## Pusat Konfigurasi (Configuration Center)

> Bagi pengguna yang menggunakan Pusat Konfigurasi (Configuration Center),
> harap dicatat bahwa bagian ini dapat diabaikan jika Anda tidak menggunakannya.

Pusat Konfigurasi telah direfaktor sepenuhnya pada versi ini, pastikan Anda
membaca kembali dokumentasi terkait dengan cermat.

Secara seragam perlu menyertakan komponen `hyperf/config-center`, perintahnya
adalah sebagai berikut:

```shell
composer require "hyperf/config-center:~2.2.0"
```

Dan sertakan komponen dependen driver yang sesuai berdasarkan driver yang
digunakan. Jika Anda menggunakan `Apollo`, Anda perlu mengimpor komponen
`hyperf/config-apollo`, dan driver lainnya serupa.

Pada saat yang sama, semua informasi konfigurasi yang berkaitan dengan pusat
konfigurasi telah dikumpulkan ke dalam `config/autoload/config_center.php`.
Silakan lakukan konfigurasi sesuai dengan struktur konfigurasi yang baru. Jika
file ini belum ada, Anda dapat menjalankan perintah `php bin/hyperf.php
vendor:publish hyperf/config-center` untuk membuatnya.

## Pusat Layanan (Service Center)

Bagi pengguna yang menggunakan komponen `hyperf/service-governance`, karena
adapter `consul` telah dipisahkan dari komponen ini, versi baru perlu
menambahkan komponen `hyperf/service-governance-consul`. Perintahnya adalah
sebagai berikut:

```shell
composer require "hyperf/service-governance-consul:~2.2.0"
```

Pengguna yang menggunakan `nacos` sebagai driver pusat layanan perlu menyertakan
komponen `hyperf/service-governance-nacos`, perintahnya adalah sebagai berikut:

```shell
composer require "hyperf/service-governance-nacos:~2.2.0"
```

## php-cs-fixer

Jika Anda tidak perlu meningkatkan `php-cs-fixer` ke versi `3.0`, Anda dapat
mengabaikan bagian ini.

1. Ubah nomor versi

```
"friendsofphp/php-cs-fixer": "^3.0"
```

2. Ubah nama file .php_cs

Ubah namanya menjadi `.php-cs-fixer.php` dan sesuaikan kode terkait
berdasarkan catatan perubahan berikut:

```diff
- return PhpCsFixer\Config::create()
+ return (new PhpCsFixer\Config())

- 'commentType' => 'PHPDoc',
+ 'comment_type' => 'PHPDoc',
```

## Penurunan kompatibilitas DI pada PHP versi 7.3

> Pengguna PHP versi yang lebih baru dapat mengabaikan bagian ini.

Pada versi `2.0` - `2.1`, untuk mengimplementasikan `AOP` agar dapat bekerja
pada objek yang tidak dikelola oleh `DI` (seperti objek yang diinstansiasi
dengan kata kunci `new`), implementasi dasarnya menggunakan komponen
`BetterReflection`. Meskipun memberikan pengalaman pemrograman baru, hal ini
juga membawa beberapa masalah yang sulit diatasi, sebagai berikut:

- startup proyek lambat tanpa cache pemindaian
- `Inject` dan `Value` tidak berfungsi dalam kasus-kasus tertentu
- `BetterReflection` belum mendukung PHP 8 (per rilis 2.2)

Pada versi baru, penerapan `BetterReflection` telah didepresiasi, dan metode
pemindaian subproses (`subprocess scanning`) digunakan untuk mengatasi masalah
di atas, namun terdapat beberapa ketidakcocokan pada versi `PHP` yang lebih
rendah:

Setelah memulai aplikasi dengan `PHP 7.3`, muncul kesalahan (error) yang mirip
dengan berikut:

```bash
PHP Fatal error: Interface 'Hyperf\Signal\SignalHandlerInterface' not found in vendor/hyperf/process/src/Handler/ProcessStopHandler.php on line 17

PHP Fatal error: Interface 'Symfony\Component\Serializer\SerializerInterface' not found in vendor/hyperf/utils/src/Serializer/Serializer.php on line 46
```

Masalah ini disebabkan karena pada `PHP 7.3`, refleksi diperoleh melalui
pemindaian subproses, dan sebuah `Interface` yang tidak ada diimplementasikan
di kelas tertentu, sehingga menyebabkan exception `Interface not found`
dilemparkan. Hal ini tidak terjadi pada `PHP` versi yang lebih tinggi.

Solusinya adalah dengan membuat `Interface` terkait dan mengimpornya dengan
normal. Solusi untuk error yang dilaporkan di atas adalah dengan menginstal
komponen dependensi yang sesuai.

> Tentu saja, akan lebih baik jika Anda melakukan peningkatan ke versi 7.4 atau
> 8.0.

```bash
composer require hyperf/signal

composer require symfony/serializer
```

## Sistem File (File System)

> Bagi pengguna yang menggunakan sistem file (file system), harap dicatat
> bahwa jika Anda tidak menggunakannya, Anda dapat mengabaikan bagian ini.

Mulai dari v2.2, Hyperf mendukung komponen `League\Flysystem` versi `v1.0`
dan `v2.0`.

Jika Anda menggunakan `League\Flysystem` versi `v2.0`, adapter yang sesuai
juga perlu disesuaikan menurut daftar berikut. Silakan merujuk ke bab
`Sistem File` untuk dokumentasi spesifiknya.

- Adapter Alibaba Cloud OSS

```shell
composer require hyperf/flysystem-oss
```

- Adapter S3

```shell
composer require "league/flysystem-aws-s3-v3:^2.0"
composer require hyperf/guzzle
```

- Adapter Qiniu

```shell
composer require "overtrue/flysystem-qiniu:^2.0"
```

- Adapter Memory

```shell
composer require "league/flysystem-memory:^2.0"
```

- Adapter Tencent Cloud COS

> Konfigurasi dasar adapter ini sedikit berbeda dari versi `v2.0` komponen
> `overtrue/flysystem-cos`. Silakan lakukan modifikasi yang sesuai berdasarkan
> konfigurasi versi terbaru.

```shell
composer require "overtrue/flysystem-cos:^4.0"
```
