# Panduan Upgrade 3.2

- Versi 3.2 terutama mengubah versi minimum `PHP` menjadi `8.2`.
- Meng-upgrade komponen `symfony/*` agar mendukung `^6.0 || ^7.0`.
- Meng-upgrade `phpunit/phpunit` ke `^11.0`.

## Perubahan Breaking

1. Komponen `async-queue` mengubah nama `JobInterface::getQueueName()` menjadi `getPoolName()`.

```php
<?php
// Sebelum
class CustomJob extends \Hyperf\AsyncQueue\Job
{
    public function getQueueName(): string
    {
        return 'custom';
    }
}

// Sesudah
class CustomJob extends \Hyperf\AsyncQueue\Job
{
    public function getPoolName(): string
    {
        return 'custom';
    }
}
```

2. Struktur konfigurasi `logger` telah berubah. Lihat [#7563](https://github.com/hyperf/hyperf/pull/7563).

3. Struktur konfigurasi `cache` telah berubah. Lihat [#7594](https://github.com/hyperf/hyperf/pull/7594).

4. Saat menggunakan `swow`, tambahkan dependency response emitter.

```php
<?php
// config/autoload/dependencies.php

use Hyperf\Contract\ResponseEmitterInterface;
use Hyperf\Engine\ResponseEmitter;

return [
    ResponseEmitterInterface::class => ResponseEmitter::class,
];
```

## Peningkatan Dependensi

- Meng-upgrade versi php ke `>=8.2`.
- Meng-upgrade `elasticsearch/elasticsearch` ke `^8.0 || ^9.0`.
- Meng-upgrade `nikic/php-parser` ke `5.6`.
- Meng-upgrade komponen `symfony/*` ke `^6.0 || ^7.0`.
- Meng-upgrade `phpunit/phpunit` ke `^11.0`.
- Meng-upgrade `google/protobuf` ke `^3.6.1 || ^4.2`.
- Meng-upgrade `guzzlehttp/guzzle` ke `^7.0`.

## Catatan

Jika Anda telah meng-upgrade ke PHP 8.4, `fgetcsv` dan `fputcsv` sekarang mewajibkan nilai default untuk parameter `escape`.

```php
fputcsv($fp, $fields, escape: '');
fgetcsv($fp, escape: '');
```
