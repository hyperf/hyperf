# Release Planning

## Version Lifecycle

| Version | Status | Active Support End Date | Security Maintenance End Date | Release or Estimated Release Date |
| :--- | :--- | :--- | :--- | :--- |
| 3.2      | Active Support  | 2027-06-07 | 2028-01-01 | 2026-06-07 |
| 3.1(LTS) | Security Support | 2026-06-07 | 2027-06-07 | 2023-12-01 |
| 3.0 | End of Life | 2023-11-30 | 2024-06-30 | 2023-01-03 |
| 2.2 | End of Life | 2022-06-20 | 2023-11-30 | 2021-07-19 |
| 2.1 | End of Life | 2021-06-30 | 2021-12-31 | 2020-12-28 |
| 2.0 | End of Life | 2020-12-28 | 2021-06-30 | 2020-06-22 |
| 1.1 | End of Life | 2020-06-23 | 2020-12-31 | 2019-10-08 |
| 1.0 | End of Life | 2019-10-08 | 2019-12-31 | 2019-06-20 |

* Active support mencakup perbaikan bug, perbaikan keamanan, iterasi fitur, dan fitur baru dalam siklus iterasi reguler;
* Security maintenance hanya mencakup perbaikan keamanan;
* Versi yang sudah End-of-Life tidak akan menerima perubahan kode apa pun. Silakan ikuti upgrade guide untuk meningkatkan ke versi terbaru agar mendapatkan dukungan yang lebih baik.

## Version Iteration Cycle

Hyperf mengadopsi model pengembangan agile, dengan rencana iterasi mingguan dan rilis `setiap hari Senin (UTC/GMT+08:00)`. Umumnya berupa rilis z-version, atau bisa juga y-version. Untuk x-version, rencana dan jadwal spesifik ditentukan berdasarkan hasil riset aktual.
Untuk aturan versi yang digunakan oleh Hyperf, silakan lihat bab [Version Description](id/versions.md).
